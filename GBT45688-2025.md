# 蔚来换电视角下的模型实施要点

## 1. 换电场景特征与监测重点


> 
**换电模式核心特征**：电池与车辆解耦、电池资产独立运营、高频换电行为、电池多车辆流转


| 特征维度 | 传统充电模式 | 蔚来换电模式 | 监测重点差异 |
| :--- | :--- | :--- | :--- |
| 电池归属 | 随车终身 | 资产独立、BaaS模式 | 电池级监测而非车辆级 |
| 补能方式 | 插枪充电 | 3-5分钟换电 | 换电前后电池状态突变检测 |
| 电池流转 | 固定单车辆 | 多车辆循环使用 | 跨车辆电池健康追踪 |
| SOC管理 | 用户自主 | 统一智能管理 | 换电柜/站端SOC调控监测 |

## 2. 换电场景下的关键监测差异点

### 2.1 电池级监测（替代车辆级）

**标准监测项的电池级映射**：

| 标准监测项 | 车辆级指标 | 换电模式映射（电池级） |
| :--- | :--- | :--- |
| 动力蓄电池温度 | 整车电池包温度 | 单个电池资产温度曲线 |
| 单体蓄电池电压 | 整车单体极值 | 该电池在各车辆的单体极值序列 |
| SOC异常 | 整车SOC跳变 | 换电前后SOC突变（如50%→90%） |
| 绝缘电阻 | 整车绝缘值 | 电池独立绝缘检测值 |

**C.11 久置过放的换电场景扩展**：


**换电电池特殊监测**：

| 状态类型 | 场景描述 | 监测要点 |
| :--- | :--- | :--- |
| 车辆搭载状态 | 电池在车辆上使用 | 标准C.11监测（离线60/90天阈值） |
| 换电站库存状态 | 电池在换电站等待分配 | 库存电池SOC保持策略（如50%±5%） |
| 运输中转状态 | 电池在不同站点间调拨 | 运输过程状态追踪（震动/温度/位置） |

**C.12 长时间聚集停放的换电场景适配**：

- 聚集定义扩展：换电站库存电池聚集 + 车辆聚集双重监测
- 500m半径调整：换电站周边车辆聚集需关联该站库存电池状态
- SOC>80%条件：换电站在满电电池（SOC>95%）库存过高时的风险预警

### 2.2 换电动作的瞬间监测

**换电前-中-后状态突变监测**：

| 阶段 | 时间窗口 | 关键监测点 | 异常判定 |
| :--- | :--- | :--- | :--- |
| 换电前 | T-5min至T | 车辆电池温度/SOC/单体电压 | 高温预警（>60℃禁止换电） |
| 换电中 | T至T+3min | 电池物理连接断开/接入状态 | 插接异常（HVIL突变） |
| 换电后 | T+3min至T+10min | 新电池SOC跳变/温度初始化 | SOC不符（非100%满电） |

**C.3 单体电池电压极差（充电）的换电适配**：

- 标准公式C.1针对充电过程计算极差
- 换电场景：新电池接入瞬间的静态极差检测（充电状态=停车充电/0x01）
- 阈值调整：换电后首次上电的静态压差 vs 充电动态压差区分判定

### 3.3 与GB/T 45688的合规要点


> 
**合规核心**：蔚来换电模式需在满足GB/T 45688-2025全量要求基础上，补充电池资产级监测维度，确保"同一电池、多车流转"场景下的安全可追溯


**特别关注点**（区别于传统车企）:

1. **电池身份唯一性**：每块电池需有独立资产ID，与车辆VIN解耦上报
1. **跨车辆历史追踪**：C.1~C.12模型计算需聚合该电池在多车辆的历史数据
1. **换电站库存监测**：C.11/C.12需覆盖"库存电池离线"状态（区别于车辆离线）
1. **换电瞬间监测**：增加换电动作前中后三阶段状态突变检测逻辑
1. **BaaS模式数据归属**：明确电池资产数据的所有权与上报责任主体

### 4.2 充电工况隐患监测（C.1-C.3）

#### C.1 动力蓄电池最高温度（充电）

**条款依据**：附录C.1

**计算逻辑**：

| 参数 | 定义 | 数据来源 |
| :--- | :--- | :--- |
| 高温阈值 V | 电池类型相关温度阈值 | 企业自定义配置 |
| 允许时长 T | 超过阈值的累计允许时间 | 企业自定义配置 |
| 允许次数 C | 超过阈值的允许次数 | 企业自定义配置 |

**风险矩阵模型**：

| 高温阈值/℃ | V₁ | V₂ | V₃ | … |
| :--- | :--- | :--- | :--- | :--- |
| 允许时长/s | T₁ | T₂ | T₃ | … |
| 允许次数/次 | C₁ | C₂ | C₃ | … |

**建议阈值**：

- 三元锂电池：V = 60℃，T = 0，C = 0（零容忍）
- 磷酸铁锂电池：V = 65℃，T = 0，C = 0（零容忍）

**判定规则**：
```
IF 温度 > V 的累计时长 > T  OR  超过次数 > C:
    输出 = 1 (异常)
ELSE:
    输出 = 0 (正常)

```

#### C.3 单体电池电压极差（充电）

**条款依据**：附录C.3

**核心计算公式**（公式C.1）：

$$r_V = \sqrt{\frac{1}{N} \sum_{j=1}^{N}(V_{max,j} - V_{min,j})^2}$$

**参数说明**：

| 符号 | 含义 | 单位 |
| :--- | :--- | :--- |
| N | 采样点个数 | - |
| j | 采样点时刻 | - |
| V_{max,j} | 第j时刻各单体电压最大值 | V |
| V_{min,j} | 第j时刻各单体电压最小值 | V |
| r_V | 极差电压均方根 | V |

**判定阈值**：

- r_V < 0.5V：输出 0（正常）
- r_V ≥ 0.5V：输出 1（异常）

### 4.4 三电温度隐患监测（C.6-C.8）

三电温度监测采用**统一统计模型**：90天运行数据清洗 → 形成n个连续运行片段 → 计算超阈值时间占比

#### C.6 驱动电机温度

**条款依据**：附录C.6

**计算公式链**：

**公式C.2（累计次数计算）**：$$f(n_i) = \begin{cases} f(n_i), & T_{i,j} \leq T_0 \\ f(n_i) + 1, & T_{i,j} > T_0 \end{cases}$$

**公式C.3（累计时间）**：$$t_i = f(n_i) \times \Delta t$$

**公式C.4（片段总时长）**：$$L_i = t_n \times \Delta t$$

**公式C.5（时间占比）**：$$\varepsilon_i = \frac{t_i}{L_i}$$

**参数说明**：

| 参数 | 含义 | 建议值 |
| :--- | :--- | :--- |
| T₀ | 驱动电机温度安全阈值 | 80℃ |
| Δt | 采样时间间隔 | 根据实际配置 |
| εᵢ | 超阈值时间占比 | 判定指标 |

**判定规则**：

- εᵢ ≤ 20%：输出 0（正常）
- εᵢ > 20%：输出 1（异常）

### 4.5 绝缘安全与采样异常（C.9-C.10）

#### C.9 绝缘电阻

**条款依据**：附录C.9

**计算步骤**：

**公式C.14（单帧判定）**：$$f(r_j) = \begin{cases} 1, & r_j \leq r_T \\ 0, & r_j > r_T \end{cases}$$

**公式C.15（累计计数）**：$$C_r = \sum_{j=1}^{N} f(r_j)$$

**公式C.16（最终判定）**：$$R = \begin{cases} 1, & C_r \geq C_T \\ 0, & C_r < C_T \end{cases}$$

**参数说明**：

| 参数 | 含义 | 建议值 |
| :--- | :--- | :--- |
| rⱼ | 第j帧绝缘值 | 实测数据 |
| rᵀ | 绝缘异常阈值 | 100Ω/V |
| f(rⱼ) | 单帧绝缘低于阈值状态 | 0/1 |
| Cᵣ | 当日绝缘低于阈值次数 | 累计值 |
| Cᵀ | 次数阈值 | 5次 |
| R | 绝缘预警算法结果 | 0=正常, 1=异常 |

---

#### C.10 电压采样异常

**条款依据**：附录C.10

**判定逻辑**：识别电压采样键合断裂时，物理位置相邻采样点电压值一高一低的数据特征。

**计算步骤**：

**公式C.17（电压极差判定）**：$$C_1 = \begin{cases} 1, & dV \geq dV_T \\ 0, & dV < dV_T \end{cases}$$

**公式C.18（物理位置相邻判定）**：$$C_2 = \begin{cases} 1, & |locV_{max} - locV_{min}| = 1 \\ 0, & |locV_{max} - locV_{min}| \neq 1 \end{cases}$$

**公式C.19（离群值相仿判定）**：$$C_3 = \begin{cases} 1, & a < \frac{V_{max}-V_{mid}}{V_{mid}-V_{min}} < \frac{1}{a} \\ 0, & \frac{V_{max}-V_{mid}}{V_{mid}-V_{min}} \leq a \\ 0, & \frac{V_{max}-V_{mid}}{V_{mid}-V_{min}} \geq \frac{1}{a} \end{cases}$$

**公式C.20（综合判定）**：$$R = C_1 \times C_2 \times C_3$$

**参数说明**：

| 参数 | 含义 | 建议值 |
| :--- | :--- | :--- |
| dV | 电压极差（Vmax - Vmin） | 实测计算 |
| dVᵀ | 电压极差预警阈值 | 0.5V |
| locVmax | 电压最大值物理位置 | BMS采样点序号 |
| locVmin | 电压最小值物理位置 | BMS采样点序号 |
| Vmax | 当日电压最大值 | 实测数据 |
| Vmin | 当日电压最小值 | 实测数据 |
| Vmid | 电压中位数 | 当日计算 |
| a | 电压离群常数 | 0.7 < a < 0.95 |
| C1/C2/C3 | 三个判定条件 | 0=不符合, 1=符合 |
| R | 算法预警结果 | 0=正常, 1=异常 |

### 4.6 SOC管理与停放监测（C.11-C.12）

#### C.11 久置过放

**条款依据**：附录C.11

**监测逻辑**：

1. **过放预警**：通过制造商提供的产品自放电速率与最后时刻荷电状态，计算电池允许静置时长。当电池系统允许静置时长低于设定阈值，且已持续1天处于下电状态，则触发久置过放预警。

**公式C.21（允许静置时长）**：$$t = \frac{F_{SOC}}{S_D}$$

**公式C.22（预警判定）**：$$R = \begin{cases} 1, & t \leq t_T \\ 0, & t > t_T \end{cases}$$

1. **长时间离线告警**：通过数据在线情况，判定车辆离线状态，识别潜在隐患。非私人购买或营运车辆离线时长不超过60天，私人车辆离线时长不超过90天。

**参数说明**：

| 参数 | 含义 | 建议值 |
| :--- | :--- | :--- |
| t | 电池系统允许静置时长 | 计算值，单位h |
| F_SOC | 电池系统数据最后时刻的荷电状况 | 实测SOC |
| S_D | 制造商提供的产品自放电速率 | SOC/d |
| t_T | 电池系统允许静置时长预警阈值 | 5天 |
| R | 算法预警结果 | 0=正常, 1=异常 |

---

#### C.12 长时间聚集停放

**条款依据**：附录C.12

**计算步骤**：

a) 判断新能源汽车当前车辆状态是否为熄火状态，并计算熄火后停放时间；b) 获取新能源汽车当前时间和当前SOC值；c) 选取所有停放时间超过60天且SOC大于80%的车辆，记为车辆集X，获取车辆集中的位置信息；d) 对于车辆集X内每一台目标车辆，计算该车半径500m内的车辆数量。

**公式C.23（距离判定）**：$$N_x = \begin{cases} 1, & d_x < 500 \\ 0, & d_x \geq 500 \end{cases}$$

**公式C.24（数量统计）**：$$Num = \sum_{x=1}^{y} N_x$$

**公式C.25（最终判定）**：$$R = \begin{cases} 1, & Num \geq Num_T \\ 0, & Num < Num_T \end{cases}$$

**参数说明**：

| 参数 | 含义 | 建议值 |
| :--- | :--- | :--- |
| N_x | 目标车辆与车辆x距离是否小于500m | 0=不符合, 1=符合 |
| d_x | 通过经纬度计算出的两车距离 | 单位m |
| Num | 目标车辆500m内停放时间超60天且SOC>80%的车辆数 | 计算值 |
| Num_T | 车辆数允许阈值 | ≤100辆 |
| R | 算法预警结果 | 0=正常, 1=异常 |

## 5. 预警规则与处置措施

依据 **GB/T 45688-2025 第6章、附录D**，建立分级预警与处置机制。

### 5.1 故障通知分级（3级故障）

| 故障等级 | 定义 | 通知对象/方式 | 处置建议 |
| :--- | :--- | :--- | :--- |
| 1级故障 | 不影响车辆正常行驶的故障 | 车载终端向车辆发送故障信息，或向车辆所有者手机推送故障信息及处置建议 | 7天内检修处理 |
| 2级故障 | 影响车辆性能，需限制车辆行驶速度、距离、路线或时间等的故障 | 同上 | 1天内检修处理 |
| 3级故障 | 驾驶人应立即停车处理或请求救援的故障 | 同上 | 停车、熄火，立即检修处理 |


> 
**批次质量预警**：同一车型车辆在统计周期内（如1年）出现3级运行安全故障的车辆数占该车型在用车辆数的 **5%以上** 时，需通过电话、电子邮件等方式向车辆生产企业相关负责人发出故障信息及处置建议，并对同批次产品进行质量核查。


---

### 5.2 隐患告警分级

| 告警条件 | 告警对象/方式 | 处置建议 |
| :--- | :--- | :--- |
| 24h内1次隐患 | 向车辆所有者手机推送隐患信息及处置建议 | 7天内维护处理 |
| 24h内≥2次隐患 | 同上 | 1天内维护处理 |


> 
**批次质量预警**：同一车型车辆在统计周期内（如1年）出现运行安全隐患的车辆数占该车型在用车辆数的 **10%以上** 时，需通过电话、电子邮件等方式向车辆生产企业相关负责人发出隐患信息及处置建议，并对同批次产品进行质量核查。
